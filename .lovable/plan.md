

## Intraday Compounding Mode + Lodgers System

A new closed-loop trading mode targeting a probabilistic 1–2% daily compounding band, governed by volatility / liquidity / reflexivity. Every trade is pre-validated by micro-horizon simulation, post-validated and logged into the **Lodgers System** — a stateful trade intelligence layer that grows a trader-specific edge graph over time.

This becomes a top-level tab in the dashboard (`Compounding`) alongside Sandbox/Augment/Risk, and adds one new edge function plus a new Supabase table.

### What the user will see

**Tab: Compounding**

1. **Daily Target Band** — header strip showing today's probability-weighted return cone (e.g. `Target 1.0–2.0% · P(hit)=62% · P(ruin)=3%`), driven by current σ, liquidity score, and reflexivity index for the active basket. This is framed as a *probability band, never a promise*.
2. **Pre-Trade Validator** — input ticker + size → runs micro-horizon scenario sim (5/15/30/60-min GBM with regime-conditional drift, slippage, reflexive impact) → returns `GO / SHRINK / SKIP` with expected hold-time, expected vs target return, edge score, and the position size the system would actually take given today's residual risk budget.
3. **Active Lodge** — live panel of open intraday positions with rolling Sharpe, realized vs expected divergence, time-in-trade vs optimal hold curve, and a pulsing "edge decay" bar. Auto-suggests exit when edge collapses below threshold.
4. **Lodger Ledger** — chronological feed where every closed trade collapses into a one-line distilled lesson (e.g. *"Held NVDA 14m past peak edge — exit discipline cost 38bps. Optimal hold for vol-regime A = 9–12m."*) generated by AI from the trade's full fingerprint + prior scars.
5. **Edge Graph** — trader-specific performance graph with four small panels:
   - Optimal hold-time distribution (histogram, with current trade marker)
   - Edge decay curve (avg realized return vs minutes held)
   - Overtrading threshold (trades/day vs avg PnL — flips red past inflection)
   - Compounding equity curve (actual vs target 1% / 2% / risk-of-ruin envelopes)
6. **Discipline Governor** — soft circuit breakers: daily loss cap, consecutive-loss cooldown, overtrading lockout, post-win cooling-off. Surfaces as a status row, not a wall.

### How it works (technical)

**New files**
- `src/components/compounding/CompoundingMode.tsx` — page shell, layout, target band, governor
- `src/components/compounding/PreTradeValidator.tsx` — input + micro-sim result UI
- `src/components/compounding/ActiveLodge.tsx` — live open-position telemetry
- `src/components/compounding/LodgerLedger.tsx` — distilled-lesson feed
- `src/components/compounding/EdgeGraph.tsx` — 4-panel Recharts dashboard
- `src/hooks/useLodgers.ts` — central state hook: load/insert/aggregate lodges, compute rolling Sharpe/Sortino, edge-decay curve, optimal-hold distribution, overtrading threshold, daily compounding curve
- `src/hooks/useIntradayValidator.ts` — runs pre-trade micro-sim (uses existing `quant-engine` for σ/μ/jump params from `useHistoricalPrices`, layered with intraday GBM + reflexive impact term + slippage from spread proxy)
- `src/lib/lodgers-math.ts` — pure math: rolling Sharpe/Sortino, drawdown elasticity, edge decay regression, overtrading inflection point (logistic fit), Kelly-capped sizing under daily-loss budget
- `supabase/functions/lodger-distill/index.ts` — given a closed-trade fingerprint + last 20 lodges + scar tags, returns one-line lesson via Lovable AI Gateway (`google/gemini-3-flash-preview`, tool-calling for structured `{lesson, tags[], pattern_id}`)

**New table** `lodger_trades` (RLS: user-scoped, mirrors `odgs_trade_ledger` policy pattern)
```
id uuid pk · user_id uuid · ticker text · side text · entry_ts bigint · exit_ts bigint
entry_px numeric · exit_px numeric · qty numeric · pnl_pct numeric · pnl_abs numeric
expected_pct numeric · expected_hold_min numeric · actual_hold_min numeric
regime text · vol_at_entry numeric · liquidity_score numeric · reflex_score numeric
exec_latency_ms numeric · slippage_bps numeric · realized_sharpe numeric
divergence_pct numeric (realized − expected) · drawdown_elasticity numeric
lesson text · tags jsonb · pattern_id text · created_at timestamptz default now()
```

**Integration points**
- New tab `compounding` added to `tabs[]` in `src/pages/Index.tsx`.
- Lodger close hook reuses `useOutcomeGradient.ingestTrade` so closed lodges also feed ODGS — Lodgers and ODGS coexist (Lodgers = behavioural/discipline graph, ODGS = profit-bias graph).
- `ScarMemory.computeScarBiases` is read by `useIntradayValidator` so scarred tickers get penalty applied to GO/SHRINK/SKIP score.
- Pre-trade validator and Active Lodge respect `useMarketRegime` for regime-conditional drift.

**Math/closed-loop**
- Pre-trade: σ, μ, jumpProb from `quant-engine` on 1y daily + last 20 intraday bars; 2,000-path GBM over chosen horizon → `P(hit target)`, `E[hold]`, `E[return]`, `CVaR_5%`. Position size = min(Kelly·0.25, residual daily loss budget / stop distance, scar-adjusted cap).
- Post-trade: full fingerprint persisted; rolling 30-trade Sharpe/Sortino; edge-decay computed as `realized_return ~ a − b·log(hold_min)` per regime; optimal hold = argmax of regression; overtrading inflection from logistic fit `pnl ~ trades_per_day`.
- Distilled lesson written back into the same row → next pre-trade call reads top-K lessons matching the candidate's regime+vol bucket and feeds them into the validator's score (feedback compression).

**Diagram**
```text
ENTRY → PreTradeValidator (μ,σ,jump,reflex,scars,daily-budget)
         │       │
         │   GO/SHRINK/SKIP + size
         ▼
     ActiveLodge (live Sharpe, edge-decay bar, exit nudge)
         │
       CLOSE
         ▼
    lodger_trades row  ──►  lodger-distill (AI one-line lesson)
         │                          │
         ├──► useLodgers aggregates (EdgeGraph, optimal hold, overtrade threshold)
         └──► ODGS ingestTrade (profit-bias)
                   │
                   └──► next PreTradeValidator call (loop closed)
```

### Guardrails (non-negotiable)
- Target band always shown as `probability range`, never as a guaranteed return.
- Risk-of-ruin and CVaR shown alongside target on the same strip.
- Discipline Governor blocks new entries when daily loss cap or overtrade lockout trips.
- All AI lessons go through `safeParseJSON`; UI degrades gracefully if `lodger-distill` returns nothing.

### Out of scope for this iteration
- Live broker order routing (uses paper/manual entries; Alpaca wiring can be added next).
- Options/futures lodges (equities + ETFs first; schema is asset-class-agnostic so extension is additive).
